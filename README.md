## рҹ‘» What is Ghost Chat?

Ghost Chat is a **single-file, serverless, end-to-end encrypted** chat app that runs entirely in your browser. Two people can talk privately with military-grade encryption вҖ” no sign-up, no data stored anywhere, no middleman.

When you close the tab, **everything is gone. Forever.**

It's not a product. It's a tool вҖ” for people who care about privacy.

## вңЁ Features

### рҹ”’ Security First
- **AES-256-GCM** app-layer encryption on every single message
- **ECDH P-256** key exchange вҖ” keys never leave your device
- **ECDSA P-256** message signing вҖ” every message is cryptographically verified
- **Ephemeral forward secrecy** вҖ” new key pair every session
- **Safety Numbers** вҖ” verify your peer out-of-band (Signal-style)
- **Replay attack protection** вҖ” sequence numbers + message ID deduplication

### рҹ’¬ Chat Features
- Real-time encrypted messaging with **optimistic delivery** (messages appear instantly)
- **Message queue** вҖ” messages are held locally and auto-delivered when connection restores
- **Burn After Read** вҖ” set messages to self-destruct after 5s / 10s / 30s / custom
- **Typing indicators** вҖ” encrypted so even your metadata is private
- **Read receipts** (вң“ / вң“вң“) with seen confirmation
- **Reply to messages** вҖ” quote any message in thread
- **Emoji reactions** вҖ” double-tap to react рҹ‘Қ вқӨпёҸ рҹҳӮ
- **Right-click context menu** вҖ” reply, copy, or delete locally

### рҹ“Ғ File Sharing
- Send **any file type** up to 25MB
- Images render inline with a click-to-expand preview
- All files are chunked and encrypted before sending

### рҹҢҗ Connection & Network
- **P2P via WebRTC** вҖ” traffic goes browser-to-browser, not through a server
- **Live latency indicator** вҖ” ping/pong heartbeats show your connection quality
- **Auto-reconnect** вҖ” peer reconnects automatically on interruption
- **Cover traffic** вҖ” randomized dummy packets make traffic analysis harder
- One-click **room code** or **invite link** sharing

### рҹҡЁ Emergency Features

## рҹ”җ How It Works

```
Person A                                    Person B
   в”Ӯ                                           в”Ӯ
   в”Ӯ  1. Generate ECDH keypair                 в”Ӯ  1. Generate ECDH keypair
   в”Ӯ  2. Generate ECDSA keypair                в”Ӯ  2. Generate ECDSA keypair
   в”Ӯ                                           в”Ӯ
   в”Ӯв”Җв”Җв”Җв”Җ public keys (via PeerJS relay) в”Җв”Җв”Җв”Җв”Җв”Җв–¶в”Ӯ
   в”Ӯв—Җв”Җв”Җв”Җ public keys (via PeerJS relay) в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ӯ
   в”Ӯ                                           в”Ӯ
   в”Ӯ  3. Derive shared AES-256 key (ECDH)      в”Ӯ  3. Derive shared AES-256 key (ECDH)
   в”Ӯ     (relay never sees this key)           в”Ӯ     (relay never sees this key)
   в”Ӯ                                           в”Ӯ
   в”Ӯв”Җв”Җв”Җв”Җ encrypted + signed message в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–¶в”Ӯ  4. Verify ECDSA signature
```

**The PeerJS relay server only brokers the initial WebRTC connection.** Once the P2P channel is open, all traffic flows directly between browsers. The relay sees encrypted WebRTC packets вҖ” it has no access to your messages, keys, or identities.

## рҹӣЎпёҸ Security Model

| Property | Details |
|---|---|
| **Transport** | DTLS 1.3 (WebRTC) |
| **App layer** | AES-256-GCM, fresh IV per message |
| **Key exchange** | ECDH P-256 (ephemeral per session) |
| **Signing** | ECDSA P-256 вҖ” every message authenticated |
| **Key derivation** | HKDF-SHA256 with domain separation |
| **Forward secrecy** | New keypair every session |
| **Message padding** | Fixed-size buckets (256 / 1KB / 4KB / 16KB) to resist length analysis |
| **Cover traffic** | Random-interval dummy packets |
| **Replay protection** | Sequence numbers + message ID dedup set |
| **Persistence** | Zero вҖ” nothing written to disk, localStorage, or IndexedDB |
| **Rate limiting** | 80 messages / 60 seconds per peer |

### What Ghost Chat does NOT protect against
- A compromised browser / device (keylogger, malicious extension)
- Screen recording or shoulder surfing
- Your ISP seeing that you're connected to PeerJS (but not the content)
- A malicious peer (the person you chose to talk to)
